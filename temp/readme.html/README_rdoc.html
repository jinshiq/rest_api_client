    Overview¶ ↑

    This application is a part of JSON API project. It shows how to retrieve user data/photos from another applciation via API with/without API authentication.

    The whole JSON API project has two applications. This application is the support one, called JSON API Support. The other one is the main application, called JSON API Main.

    JSON API Main creates user data/photos and APIs, while JSON API Support retrieves user data/photos from JSON API Main via APIs with/without authentication.

    JSON API Main creates two APIs, one for User model and the other for Photo model. User model has text fields only and is designed to require no API authentication for API access, while Photo model contains image field and is designed to require API authentication.

    This application uses 'httparty' gem to parse JSON API and uses 'figaro' gem to hide sensitive information such as API_KEY when accessing third-party's API (i.e. JSON API Main API in our case).

    The source code is available here at github.

    Access JSON API¶ ↑

    If API authenticaion is not requred, e.g. User api in our case, set url like below.

    url = "https://aqueous-ocean-76425.herokuapp.com/users.json"

url = "https://aqueous-ocean-76425.herokuapp.com/users/#{params[:id]}.json"

    If API authenticaion is requred, e.g. Photo api in our case, set url like below.

    That means a valid API_KEY should be passed to url for authentication. Such API_KEY is stored at /config/application.yml which is created by 'figaro' gem, so API_KEY cannot be seen by other people or from browser logs.

    url = "https://aqueous-ocean-76425.herokuapp.com/api/v1/photos.json?token=#{ENV['API_KEY']}"

url = "https://aqueous-ocean-76425.herokuapp.com/api/v1/photos/#{params[:id]}.json?token=#{ENV['API_KEY']}"
